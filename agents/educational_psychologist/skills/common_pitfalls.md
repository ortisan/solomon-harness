# Educational Psychologist Common Pitfalls

This skill lists common pitfalls to avoid when designing learning architectures and instructional content.

## Common pitfalls

- Using unscientific learning styles to design instruction, which has been shown to be ineffective by empirical research.
- Presenting too much information at once, leading to cognitive overload and decreased learning efficiency.
- Creating assessments that do not match the learning objectives, which fails to measure actual student understanding.
- Relying on passive review rather than active recall, which reduces long-term retention.
- Presenting redundant visual and auditory information, which increases cognitive load.

## Definition of done

- [ ] Every design decision cites a methodology from the closed allow-list (Cognitive Load Theory, retrieval practice, distributed practice, dual coding, backward design); no learning-styles or other rejected-fad justification appears anywhere.
- [ ] No screen or unit of instruction exceeds the four-chunk working-memory limit; longer material is segmented and sequenced.
- [ ] Each assessment task exercises the exact behavior in its measurable learning objective, preserving constructive alignment.
- [ ] The design schedules active recall prompts at least every six minutes of content and never relies on passive re-reading as the primary review method.
- [ ] No diagram carries on-screen text that repeats a simultaneous narration word for word; text is integrated into the visual or moved to the audio channel.
- [ ] The review notes rule out each pitfall above explicitly before the deliverable is accepted.
