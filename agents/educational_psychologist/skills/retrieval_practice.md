# Retrieval Practice

This skill governs the incorporation of active recall, self-testing, and the testing effect into learning designs to promote long-term retention and schema consolidation.

## Theory and Background
Retrieval practice is the act of actively calling information to mind from memory. According to research on the retrieval practice / testing effect (Roediger and Karpicke), testing is not merely a tool for assessment, but a powerful learning event. Actively retrieving information makes that information more retrievable in the future. In contrast, passive study methods, such as re-reading text or re-watching videos, create a false sense of fluency and result in rapid forgetting.

When a learner retrieves information, the neural pathways associated with that memory are strengthened, making it easier to access later. Furthermore, retrieval practice helps identify gaps in knowledge, allowing learners to focus their efforts on areas that need reinforcement. The effortful nature of retrieval is a key component; more difficult retrieval leads to greater memory enhancement, provided the retrieval is successful. This phenomenon is known as desirable difficulty.

The testing effect has been demonstrated in numerous studies across various age groups and subject domains. It shows that testing leads to better long-term retention than an equivalent amount of additional study time, even when no feedback is provided. However, when feedback is provided, the benefits of retrieval practice are even greater, as it allows learners to correct errors and consolidate correct information.

## Design Standards and Concrete Guidelines
To implement retrieval practice effectively, learning designs must incorporate frequent opportunities for active recall. We establish a concrete standard: learners must be prompted with an active recall prompt or quiz at least once every six minutes of reading, watching, or listening. This ensures that learners do not engage in passive consumption for extended periods and are forced to retrieve and process the information they have just encountered.

Quizzes should not be high-stakes; they should be low-stakes or no-stakes formative assessments designed to support learning. Feedback must be provided immediately after retrieval to correct errors. When designing multiple-choice questions, distractors must be plausible but clearly incorrect to ensure the retrieval is meaningful and effortful.

### Worked Example: Interactive Tutorial Design
In an interactive programming tutorial, do not simply present a code snippet and ask the user to copy and paste it. Instead, present the code snippet, explain its function, and then hide the snippet. Ask the user to write the code from memory to achieve a specific outcome. The system must validate the user's input and provide immediate feedback within two seconds of submission.

If the user fails to retrieve the correct code, the system should provide a subtle hint (such as the first character of the command) rather than displaying the entire solution immediately. This maintains a level of effortful retrieval while preventing frustration. Once the correct code is successfully retrieved, the system should present a brief explanation of why the code is correct to consolidate the schema.

## Common pitfalls
- Relying on passive re-reading or highlighted text review as the primary study recommendation.
- Delaying quizzes or recall prompts until the end of a long unit, allowing significant forgetting to occur beforehand.
- Designing recall questions that are too easy, failing to create the effortful retrieval necessary for the testing effect.
- Failing to provide immediate feedback after a retrieval attempt, leaving errors uncorrected.
- Making retrieval opportunities high-stakes, which increases anxiety and distracts from the learning process.
