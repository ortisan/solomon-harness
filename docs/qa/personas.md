# Personas

Personas frame every exploratory session: a session is one persona pursuing one
journey, judged by whether that persona reaches a true end state. Derive
project-specific personas from the product's real user types; the entries below
are the seed catalog to extend, not a fixed list.

## New User

First contact with the product, no accumulated context. Goal: complete the
primary job without reading documentation. Sensitive to unexplained states,
dead ends, and jargon. Abandons on the first step that offers no obvious next
action.

## Returning User

Has done the primary job before and expects it to work the same way. Goal: speed
and no surprises. Sensitive to changed layouts, moved controls, and silent
behavior changes. Notices when a previously-`pass` scenario now behaves
differently.

## Recovering User

Arrives after something went wrong (an error, an interruption, a stale session).
Goal: understand what state they are in and get back on track. Sensitive to
missing or misleading error messages, lost work, and irreversible actions with no
confirmation.

## Operator

Runs the system rather than uses its product surface (the harness's own
maintainers, in this repo). Goal: observe status, recover from failure, and trust
the ledger. Sensitive to a green matrix over a red suite, a stale claim, and a
run reported as `ok` that delivered nothing.
